# Visio Divina 專案開發規範

> [!IMPORTANT]
> **分支策略與推送規範**
> 1. **預設開發分支**：從現在起，所有的代碼修改與 git push 預設都只能推送到 `dev` 分支。
> 2. **自動提交與推送 (Dev)**：在 `dev` 分支下，每次修改後必須自動 Commit (包含 [skip ci]) 並 Push。完成後提醒使用者前往預覽網址：https://dev.visiodivina.pages.dev
> 3. **發布與合併 (Main)**：嚴禁自動合併到 `main` 分支。只有當使用者明確指令（如：「發布」、「正式上線」或「Merge 到 main」）時，才執行合併並推送到 `main` 分支。
> 4. **全域規則例外**：本專案規則優先於全域規則。

## 其他注意事項
- 地圖標籤 (Map Labels) 應保持純文字高對比顯示，無背景方框。
- 專注模式標題應動態顯示書卷名稱。
